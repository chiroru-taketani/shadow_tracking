# LiDARによる物体検出・トラッキングプログラム (`main_cv.cpp`)

このプログラムは、URGセンサー（LiDAR）を用いて前方の物体（主に人や足など）をスキャンし、最も近くにある物体の現実世界の座標をリアルタイムに計算して外部ファイルに出力するC++アプリケーションです。

## 概要

1. **LiDARパラメーターの取得**  
   URGセンサーから得られた距離（mm単位）と角度データをもとに、各スキャン点の座標を算出します。
2. **OpenGLを用いた仮想的な描画**  
   得られた座標点を、OpenGLを用いて2D平面上に描画します（各点にはテクスチャ `mark.png` を貼り付けます）。距離に応じて描画サイズを調整し、点群の隙間を埋める工夫がされています。
3. **OpenCVを用いた画像解析**  
   OpenGLの描画バッファ（画面内容）をOpenCVの画像フォーマット（`cv::Mat`）として読み込みます。画像に対して2値化およびモルフォロジー変換（膨張・収縮処理）を行い、個別の点群を一つの塊（輪郭）として結合させます。
4. **対象物の抽出と座標変換**  
   画像上で検出された複数の輪郭のうち、一定の面積（ピクセル）以上のものを対象とします。それぞれの重心を求め、再度OpenGLの座標逆変換関数（`gluUnProject`）を用いることで画面上のピクセル座標から現実世界の座標（cm単位）へと逆変換します。
5. **最接近物体の保存**  
   検出できた物体の中で、LiDARセンサーに「最も近い」対象のみを一つピックアップし、その重心座標を `footpoint.txt` にリアルタイム出力します。

## 座標認識・変換のロジック（詳細）

本プログラムでは、ハードウェアからの生データを正確に現実空間の座標（cm単位）として取り扱うため、幾つかのステップを経て座標の変換（順変換および逆変換）を行なっています。

### 1. 極座標（センサー基準）から直交座標（OpenGL空間）への変換
LiDARから取得されるデータは極座標系（距離と角度）です。
*   取得した距離 `distData` (mm) はスケール係数 `0.1` を掛けて **cm単位** に変換します（距離 `len = distData * 0.1`）。
*   スキャン点のインデックスをラジアンの角度 `rad` に変換します。
*   極座標からXY直交座標に変換します。
    *   `X = -len * sin(rad)`
    *   `Y = len * cos(rad)`
*   この $(X, Y)$ はセンサー位置を原点 $(0,0)$ とした座標です。この座標に対して点のテクスチャを描画し、画面を構成します。距離が離れるほど点同士の隙間が広がるため、距離の平方根に比例して描画する円のサイズを大きくし補正しています。

### 2. OpenGL座標空間の構築（順変換）
`reshape` 関数内で `glOrtho` 関数を使用し、現実空間のスケールをそのままプロジェクション（投影）空間として設定しています。
```cpp
// 幅 -scanAreaW*0.5 〜 scanAreaW*0.5、高さ 0.0 〜 scanAreaH（cm）をマッピング
glOrtho(-scanAreaW * 0.5, scanAreaW * 0.5, 0.0, scanAreaH, -100, 100);
```
これにより、画面内の仮想座標系が「現実のセンチメートル寸法」と一致するように調整されています。

### 3. OpenCVのピクセル座標空間と逆変換（抽出）
画面を再描写したら、`glReadPixels` 関数で描画されたバッファをピクセルデータとしてOpenCVの行列（`cv::Mat`）にそのまま取り込みます（OpenCVとOpenGLはy軸の上下が逆転するためフリップ処理がかかります）。
画像処理を通じてノイズを除去した後、以下の流れでピクセルから現実世界へ逆算します。

1.  **重心計算**: `cv::moments` を用いて、各物体の輪郭内のピクセル群の重心 `(p.x, p.y)` を特定します。
2.  **座標逆変換 (UnProject)**: OpenCVの画像上の重心ピクセル座標を、再び現実の座標(cm)に引き戻します。これを行うのが `gluUnProject` 関数です。
    ```cpp
    gluUnProject(p.x, binImage.rows - p.y, 0, model, proj, view, &pointF.x, &pointF.y, &pointF.z);
    ```
    OpenGLがレンダリング時に使用したビューポート(`view`)や行列(`model`, `proj`)を元にして、ピクセル座標から元の**現実座標 (pointF.x, pointF.y) （単位: cm）**にマッピングし直します。

### 4. 最接近点の判別
逆変換した対象物の座標群それぞれについて、原点（センサー位置：0,0）からのユークリッド距離を算出します。
```cpp
double distFromSensor = sqrt(pow(pointF.x, 2) + pow(pointF.y, 2));
```
この距離が最も短いもの（一番手前の物体）を特定し、その重心の現実座標を出力します。


## 実行環境・依存ライブラリ

*   **C++11** コンパイラ
*   **OpenCV (v4系)**: 画像の二値化、ノイズ除去、輪郭検出、重心計算に使用。
*   **OpenGL / GLUT**: 座標変換と、2Dテクスチャベースのスキャン点描画に使用（Mac環境の場合はフレームワークとしてリンク）。
*   **urg_cpp**: 北陽電機製URGセンサーの公式C++ドライバライブラリ (`Urg_driver.h`, `Connection_information.h` など)。

## コンパイルコマンド

Mac環境などでの標準的なコンパイルコマンドの例：
```bash
g++ -O3 main_cv.cpp -std=c++11 `pkg-config --cflags --libs opencv4` -framework OpenGL -framework GLUT Connection_information.cpp liburg_cpp.a -Wno-deprecated
```

## 入出力ファイル

*   **入力ファイル**
    *   `scanarea.txt` : スキャン領域の幅(cm), 高さ(cm), 解像度(1画素あたりのcm)が記載されているファイルです。
    *   `mark.png` : OpenGLで各スキャン点を描画する際に使用するテクスチャ画像です。
*   **出力ファイル**
    *   `footpoint.txt` : ファイルの1行目に検出した物体数（最前面の1つのみなので通常 `1` もしくは `0`）、2行目以降に対象物のX, Y座標（cm）がリアルタイムに上書き保存されます。

## プログラムの主な設定値・閾値

ソースコード内にある以下の変数を調整することで、動作を変更できます。

*   `DATASIZE` (例: 513) : 使用するURGセンサーのモデルに応じたデータ点数。
*   `scanAngle` (160) : スキャンする角度の範囲。
*   `scanAngleStep` (0.35) : センサーの1ステップあたりの角度。使用するセンサー（UBG-04LX-F01 など）に合わせて変更が必要です。
*   `objectSize` (800) : 対象として認識する最小面積閾値（ピクセル単位）。
*   `moveThreshold` (0.1) : 最小移動距離の閾値（現在未使用または拡張用）。

## 実行中の操作

*   実行中は、2値化された後の処理画像を示す `binImage` ウィンドウと、トラッキング結果（青い輪郭線と赤い重心点）が描画された `Scan` ウィンドウが表示されます。
*   終了する場合はキーボードの `ESC` キーを押してください。
