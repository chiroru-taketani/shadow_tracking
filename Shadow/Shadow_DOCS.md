### ウィンドウ

| ウィンドウ名 | 名称 | 役割 |
| --- | --- | --- |
| CG0 | 光源視点 | 光源から見た視点をレンダリングして，影の場所を特定 |
| CG1 | メイン操作画面 | 床を真上から見た視点．CG０からの計算結果をもとに，ここで影を動かす計算をする． |
| CG2 | LED出力画面 | 実際のLEDパネルに表示するための，光源の位置を描写 |

### ライブラリ

- OpenGL/GLUT：３Dグラフィックスを描画するため
- OpenCV(cv:Mat)：描画した画像をデータとして読み取りどこが影か判定する
- シリアル通信(termios.h)：計算した光源の位置を，USB経由でLEDパネルに送るための通信手段

### メインロジック

影を触ったら何故ライトが動くのかの計算

**影判定の仕組み**

1. display0で光源からの視点をレンダリング．
2. glReadPixelsを使い，その結果を画像データで取得する
3. display1で，その画像を「床」に投影
4. ユーザが床を触った時，その場所のピクセルが「黒（影）」なら「影に触れた」と判定

**幾何学的な計算**

影を動かすためのベクトル演算

- 光線ベクトル（lightVec)：影の地点と物体の位置を結ぶ線
- 逆算：影の位置を動かすと，その延長線上にある光源（lightPos0）の座標を再計算

### 各関数の役割解説

**初期化プロセス**

- `main()`：プログラムの入り口．ウィンドウを作りシリアル通信を開始
- `initGL()`：OpenGLの設定（テクスチャの読み込みや色の設定）
- `initSerial()`：ESP-32との通信するための準備

**描画プロセス（無限ループ）**

- `display0()`：光源からの視点，影の形を決定
- `display1()`：床の描画．ここにLiDARから取得した足の位置や計算された影を表示．
- `display2()`：制御用．光源の位置を2Dの点として描画
- `scene()`：３D空間に置く「物体」を描く共通関数

**インタラクション**

- `timer0()`：30fpsで以下を繰り替えす．
    1. footpoint.txt（LiDARのデータ）を読み込み，位置を把握
    2. 認識された位置（手の位置or足の位置）が影に入っているか判定
    3. 影を動かす計算を行い，新しい光源の位置を決める
    4. シリアル通信でLEDパネルに座標を送信
- `mouse1()`/`motion1()`：マウスで影をドラッグした時の処理（デバック用）

### 重要な変数・定数

- lightPos0：光源のXYZ座標．
- objPos：影を落とす物体の位置．
- footPoint：LiDARで検出した足の座標
- ObjectFlg：キーボードの「１」「２」「０」で，影を作る物体を切り替えるフラグ

### まとめ　処理の流れ

1. **入力**：LiDARが「足の位置」をテキストファイルに書き出す．
2. **判定**：timer0がそのファイルを読み，OpenGLの画像データと照らし合わせて「影の中か」チェック
3. **計算**：影を掴んで動かしたベクトルをもとに，光源の新しい位置を算出
4. **出力**：新しい座標をシリアル通信でESP-32に送り同時にPC画面上の３D表示を更新

## 操作説明

### 起動と画面構成

- **CG0 (Light View):** 光源（LED）から物体を見た視点です。ここで「影になる領域」を計算しています。
- **CG1 (Main/Floor View):** 床面を俯瞰したメイン画面です。インタラクション（影の操作）は主に行われます。
- **CG2 (LED Panel Output):** 物理的なLEDパネルに送る映像のプレビューです。

### キーボード操作

| **キー** | **動作** |
| --- | --- |
| **`1`** | 影を作る物体を**「立方体」**に変更します。 |
| **`2`** | 影を作る物体を**「立方体と球体」**に変更します。 |
| **`0`** | 影を作る物体を**「板ポリゴン（テクスチャ付き）」**に変更します。 |
| **`d`** | **デバッグモードの切替**。光源、衝突点、足の位置などを示す球体の表示/非表示を切り替えます。 |
| **`t`** | **光源位置のリセット**。ライトを初期位置（中央付近）に戻します。 |
| **`r`** | **カメラ視点のリセット**。CG1の俯瞰カメラを真上の位置に戻します。 |
| **`f`** | **全画面表示**。実行中のウィンドウをフルスクリーンにします。 |
| **`ESC`** | プログラムを**終了**します。 |

## グローバル変数について

### AppConfig　プログラム全体の設定

```cpp
struct AppConfig {
    int dispMode = 0;       // 0:通常表示, 1:デバッグ用ガイド表示（d）
    int objectFlg = 1;      // 描画する物体の種類 (0:テクスチャ板, 1:立方体, 2:立方体と球体)（数字キー）
    double frameRate = 30.0; //フレームレート
    double renderScale = 1.0; //解像度調整用のスケール係数
} g_appConfig;
```

    | **状態** | **影響** |
    | --- | --- |
    | **役割** | GPUの物理ピクセルとプログラムの論理座標のズレを吸収する。 |
    | **1.0 の場合** | 通常のディスプレイ（1ピクセル＝1ドット）に最適化。 |
    | **2.0 の場合** | Retinaディスプレイ等、高密度な描画バッファから正確に画像を抜き出す。 |

### AreaConfig  空間・エリアの設定

```cpp
struct AreaConfig {
    double scanW = 400.0; // スキャンエリアの横幅 (mm)
    double scanH = 360.0; // スキャンエリアの縦幅 (mm)
    double aspectRate = 1.0 / 2.0; // LEDパネルのアスペクト比 (縦/横)
    double lightW = 256.0;  // LEDパネルの横幅
    double lightH = 128.0; // LEDパネルの縦幅 (lightW * aspectRate)
    double resolution = 2.0; // 1mmあたりのピクセル数。画像処理の精度に影響します
} g_areaConfig;
```

- resolution
    - 物理空間（mm単位）」と「画像空間（ピクセル単位）」のスケーリング係数（解像度）
    - **働き:** 実世界の 1mm を、画面上の何ピクセルとして描画するかを定義
    - **影響:** この値を大きくすると、影の境界線がより精細になり、影判定（`shadowAreaGrayImage`）の精度が上がりますが、計算負荷（ビデオメモリの消費）も増加
    - CG1のウィンドウサイズが大きくなる
