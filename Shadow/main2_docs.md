# Shadow/main2.cpp ドキュメント

## 概要
`Shadow/main2.cpp` は、Shadow Tracking System の中核となる視覚効果とインタラクションを担当するプログラムです。
LiDARセンサーから取得した人の位置情報に基づき、仮想的な「影」と「光源」をシミュレーションし、その結果を映像として投影およびLEDパネルへ出力します。

## 主な機能
1.  **3つのビューポート描画**:
    *   **CG0 (Main View)**: ユーザーに提示するメインの映像（オブジェクトと影）。
    *   **CG1 (Shadow Map)**: 影生成のための計算用ビュー（光源からの視点）。影領域の抽出にも使用されます。
    *   **CG2 (Light Status)**: 光源の現在の位置と色状態を可視化するデバッグ/モニタリング用ビュー。
2.  **インタラクション処理**:
    *   LiDARデータ (`../LIDAR1b/footpoint.txt`) を読み込み、人の位置に連動して光源やオブジェクトを操作します。
    *   影に「触れる」ことで光源を動かすインタラクションロジックを含みます。
3.  **外部連携**:
    *   **シリアル通信**: 計算された光源位置をLEDマトリクスパネル制御用マイコン（ESP32等）に送信します。

## コンパイルと実行
ソースコード冒頭のコメントにある通り、以下のコマンド等でコンパイルします（環境依存あり）。

```bash
g++ -O3 -std=c++11 main2.cpp -framework OpenGL -framework GLUT `pkg-config --cflags --libs opencv4` -Wno-deprecated
```

## データ構造

### 設定構造体
プログラムの挙動を制御する主要なグローバル設定です。

| 構造体 | 変数名 | 説明 |
| :--- | :--- | :--- |
| `AppConfig` | `g_appConfig` | アプリケーション全体の設定（デバッグ表示フラグ、描画オブジェクト切り替え、フレームレートなど）。 |
| `AreaConfig` | `g_areaConfig` | スキャンエリアやLEDパネルの物理サイズ、解像度スケールなどの空間設定。 |
| `SerialConfig` | `g_Serial` | シリアルポート名、ボーレート、ファイルディスクリプタなどの通信設定。 |
| `InteractionConfig` | `g_interaction` | インタラクションの閾値（距離など）の設定。 |
| `ObjectConfig` | `g_object` | 描画するオブジェクトのスケール設定。 |

### 状態管理
| 構造体/変数 | 説明 |
| :--- | :--- |
| `WindowInfo` (`g_winInfo[3]`) | 3つのウィンドウ (CG0, CG1, CG2) それぞれのID、サイズ、マウス状態を管理。 |
| `Camera` (`g_Cam[2]`) | 視点（カメラ）の距離や角度情報。 |
| `Vec_3D` | 3次元ベクトルおよびテクスチャ座標(s,t,u)を扱う構造体。 |

## 主要なグローバル変数

*   `lightPos0`: 光源の現在位置 (mm)。この座標が変化することで影が動きます。
*   `touchPos`, `touchPos0`: LiDARまたはマウスから入力された「接触位置」（人の足元位置）。
*   `objPos`: 影を落とす対象となるオブジェクト（板、立方体など）の位置。
*   `frameImage`, `shadowAreaImage`: OpenCVで処理される画像バッファ。`shadowAreaImage` は「影の領域」を判定するために使用されます。

## 関数詳細

### 1. 初期化
*   **`initGL()`**:
    *   3つのGLUTウィンドウ（CG0, CG1, CG2）を作成し、それぞれのコールバック関数（描画、リシェイプ、入力）を登録します。
    *   テクスチャの読み込み (`Obj_01.png`) とOpenGL設定（デプステスト、ブレンド、テクスチャ自動生成など）を行います。
*   **`initSerial(const char *portName)`**:
    *   指定されたシリアルポートを開き、ボーレート115200bpsで通信を確立します。

### 2. メインループ
*   **`timer0(int value)`**:
    *   システムの中枢となるループ関数です。`glutTimerFunc` により定期的に呼び出されます。
    *   **処理順序**:
        1.  全ウィンドウの再描画リクエスト (`glutPostRedisplay`)。
        2.  LiDARデータの更新とインタラクション計算 (`updateLidarInteraction`)。
        3.  シリアルデータ送信 (`sendLightPosToSerial`)。
        4.  次回のタイマー予約。

### 3. ロジック / インタラクション
*   **`updateLidarInteraction()`**:
    *   共有ファイル `../LIDAR1b/footpoint.txt` を読み込みます。
    *   取得した座標をミリメートル単位 (`touchPos0`) および画像座標 (`imgX`, `imgY`) に変換します。
    *   **影判定**: `shadowAreaGrayImage` の画素値を参照し、現在の位置が「影の中 (`shadowVal0 == 0`)」にあるかを判定します。
    *   **追従ロジック**: 影の中にいる場合、光源位置 `lightPos0` を計算・更新して、影が人を追従したり逃げたりするような挙動（インタラクション）を生成します。
*   **`sendLightPosToSerial()`**:
    *   現在の光源位置 `lightPos0` をLEDパネル上の座標 (`serial_x`, `serial_y`) にマッピング変換します。
    *   `mapRange` 関数を使用。
    *   `"x,y\n"` の形式でシリアルポートへ書き込みます。

### 4. 描画
*   **`display0()` (Main View)**:
    *   ユーザーが見るメインのシーンを描画します。
    *   `scene()` 関数を呼び出し、オブジェクトを描画します。
    *   描画結果を `glReadPixels` で取得し、OpenCVの `frameImage` に格納（テクスチャ生成や画像処理用）。
    *   デバッグモード(`d`キー)時は、認識範囲枠などをワイヤーフレーム表示します。
*   **`display1()` (Shadow Calculation)**:
    *   光源からの視点でシーンを描画します。
    *   **射影テクスチャマッピング**: `display0` で生成された映像をテクスチャとして、床面に投影することで「影」を表現します。
    *   描画結果を `shadowAreaImage` として取得し、グレースケール変換して影判定に使用します。
*   **`display2()` (Light Monitor)**:
    *   光源を上空から見たような2Dビューを描画します。
    *   光源の色や位置を円で表示し、デバッグに役立てます。
*   **`scene()`**:
    *   実際に描画される3Dオブジェクト（テクスチャ板、立方体、球体など）の定義です。
    *   `g_appConfig.objectFlg` によって描画内容が切り替わります（キーボード `0`, `1`, `2` で変更可能）。

### 5. 入力操作
KEY操作によりデバッグ機能やモード切替が可能です。
*   `d`: デバッグ表示モードの切り替え。
*   `0`, `1`, `2`: 描画オブジェクトの切り替え（板、立方体、複合）。
*   `r`: カメラアングルリセット。
*   `t`: 光源位置リセット。
*   `f`: フルスクリーン表示。
*   `ESC`: 終了。
*   マウス右ドラッグ: 光源移動 (CG0) やカメラ回転 (CG1)。
*   マウス左クリック: タッチ位置のシミュレーション（デバッグ用）。

## 依存ライブラリ
*   **OpenGL / GLUT**: グラフィックス描画およびウィンドウ管理。
*   **OpenCV**: 画像処理（テクスチャ読み込み、およびレンダリング結果の解析）。
