# Shadow/main2.cpp ドキュメント

このドキュメントでは、`Shadow/main2.cpp` のコード構造と機能について解説します。
このプログラムは、OpenGLとOpenCVを使用して、仮想空間内の物体と影をレンダリングし、LIDAR入力やマウス操作に基づいて光源位置を動的に制御するシステムです。

## 概要

このプログラムは「Shadow Tracking」プロジェクトのコアとなる視覚化および制御ソフトウェアです。
主な機能は以下の通りです：

1.  **3つのOpenGLウィンドウによる描画**:
    *   **CG0 (ユーザー視点)**: ユーザーに提示するメインの映像。物体と、光源によって計算された影を表示します。
    *   **CG1 (影処理用)**: 影の生成計算を行うための内部的な視点。ここで生成された映像を解析して、影の位置や形状を認識します。
    *   **CG2 (照明制御用)**: LEDパネルなどの光源装置に出力するための情報を可視化します。
2.  **LiDARデータとの連携**:
    *   外部プログラム（LIDAR1bなど）が出力した座標データ（`../LIDAR1b/footpoint.txt`）を読み取り、ユーザーの位置（足元など）として認識します。
3.  **インタラクティブな影の制御**:
    *   ユーザーが影に触れた（影の領域に入った）ことを検知し、光源の位置を動かすことで、影があたかも物体であるかのようなインタラクション（影を踏むと逃げる、など）を実現します。
4.  **シリアル通信**:
    *   計算された仮想光源の位置を、物理的な照明装置（LEDパネルなど）にシリアル通信で送信し、現実空間の照明と同期させます。

## システム要件 / 依存ライブラリ

*   **OpenGL / GLUT**: 3Dグラフィックス描画およびウィンドウ管理。
*   **OpenCV**: 画像処理（テクスチャ読み込み、フレームバッファの解析、影の検出）。
*   **Standard C/C++ Libraries**: ファイル入出力、数学関数など。
*   **POSIX Serial**: シリアル通信用 (`termios.h`, `fcntl.h` など)。

## 主要なデータ構造

### 1. 設定構造体
プログラムの動作パラメータを管理する構造体です。

*   **`AppConfig`**: アプリケーション全体の設定。
    *   `dispMode`: デバッグ表示のON/OFF。
    *   `objectFlg`: 描画するオブジェクトの種類（0:テクスチャ板, 1:立方体, 2:複合）。
    *   `renderScale`: 解像度係数。
*   **`AreaConfig`**: 物理的な空間やデバイスのサイズ設定。
    *   `scanW`, `scanH`: スキャンエリアのサイズ (mm)。
    *   `lightW`, `lightH`: LEDパネルのサイズ。
    *   `resolution`: 画像処理の解像度（mm/pixel）。
*   **`SerialConfig`**: シリアルポート設定。
    *   ポート名（デフォルト: `/dev/cu.usbserial-10`）やボーレート。

### 2. 状態管理変数
*   **`WindowInfo`**: 各ウィンドウ（CG0, CG1, CG2）のサイズやマウス状態を管理。
*   **`Vec_3D`**: 3次元座標およびテクスチャ座標を持つベクトル構造体。
*   **`chaseFlg`**: 追従モードの状態フラグ（0:待機, 1:追従中）。
*   **`shadowVal`**: 影の検出値（0が影領域）。

## 関数の詳細解説

### メインルーチン
*   **`main()`**:
    *   GLUTの初期化および `initGL()` の呼び出し。
    *   シリアルポートのオープン (`initSerial`)。
    *   `glutMainLoop()` でイベントループを開始。
*   **`timer0()`**:
    *   システムのメインループとして機能します（約30fpsで動作）。
    *   **処理の流れ**:
        1.  全ウィンドウの再描画要求 (`glutPostRedisplay`)。
        2.  `updateLidarInteraction()`: LIDARデータの読み込みとインタラクション計算。
        3.  `sendLightPosToSerial()`: 結果をシリアル送信。
        4.  次のタイマーをセット。

### 描画・画像処理 (Display Functions)
*   **`display0()` (CG0)**:
    *   ユーザーが見るメイン画面。
    *   `scene()` 関数でオブジェクトを描画。
    *   描画結果を `glReadPixels` で取得し、OpenCVの `cv::Mat` (`frameImage`) に変換。
    *   アルファチャンネル処理を行い、影生成用テクスチャのもととなる画像 (`frameImage1`) を作成。
*   **`display1()` (CG1)**:
    *   影計算用の画面。
    *   **テクスチャ投影**: `display0` で生成した `frameImage1` を、光源位置（`lightPos0`）からの投影として床面にマッピングします（`glTexGendv`, `glMatrixMode(GL_TEXTURE)` などを利用した射影テクスチャマッピング）。
    *   これにより、床面に落ちる「影」が描画されます。
    *   描画結果を `glReadPixels` で取得し、`shadowAreaGrayImage` (グレースケール画像) として保存。これが後の影判定に使用されます。
*   **`display2()` (CG2)**:
    *   LEDパネル制御用の画面。
    *   現在の光源位置 `lightPos0` を可視化します。

### インタラクションロジック
*   **`updateLidarInteraction()`**:
    *   `../LIDAR1b/footpoint.txt` から座標を読み取ります。
    *   読み取った座標を画面上のピクセル座標に変換し、`shadowAreaGrayImage` を参照してその位置が「影（黒い部分）」かどうかを判定します。
    *   **Phase A (キャッチ)**: 影の中に手/足が入った瞬間、追従モード(`chaseFlg=1`)に移行。光源制御の基準点を設定。
    *   **Phase B (追従)**: 手/足が動くと、それに合わせて光源位置 `lightPos0` を更新し、影が逃げる（あるいは追従する）ような動きを生成します。
*   **`scene()`**:
    *   描画対象の3Dオブジェクトを定義します（テクスチャ付き平面、立方体、球体など）。
    *   デバッグモード(`dispMode=1`)時は、光源位置やタッチ位置、LIDAR検知点を球体として可視化します。

### ユーティリティ
*   **`glMyTexPlane`**: テクスチャを貼り付けた平面（板）を描画するヘルパー関数。
*   **`vectorNormalize`, `normcrossprod`**: ベクトル演算関数。
*   **`initSerial`, `sendLightPosToSerial`**: シリアル通信のセットアップとデータ送信。送信データは `x,y\n` 形式。

## ビルドと実行方法

ファイルの冒頭にコンパイルコマンドの例が記載されています：

```bash
g++ -O3 -std=c++11 main2.cpp \
    -framework OpenGL -framework GLUT \
    `pkg-config --cflags --libs opencv4` \
    -Wno-deprecated
```

### 注意点
*   `../LIDAR1b/footpoint.txt` が存在しない場合、LIDAR連携は機能しません（読み込みエラー処理あり）。
*   シリアルポート名（`/dev/cu.usbserial-10`）は環境に合わせて変更する必要があります。
